Ubuntu on EC2 — Quick Deployment Guide

Objective: Launch, secure, connect and run basic commands on an Ubuntu EC2 instance. Includes commands you can copy-paste.

Prerequisites

  • An active AWS account with EC2 permissions.
  • Familiarity with Linux shell and SSH.
  • Internet-accessible machine and public IP for the EC2 instance.
  • Optional: AWS CLI installed and configured (aws configure).

High-level steps

  1. Create a Key Pair (console or CLI).
  2. Create / configure a Security Group (open ports: 22, 80, 443).
  3. Launch an Ubuntu EC2 instance using an Ubuntu AMI.
  4. SSH into the instance using the .pem file.
  5. Run basic Linux commands and (optionally) install Nginx or Apache.

1) Create Key Pair

Console (recommended for beginners)

In the EC2 console: Key Pairs > Create key pair → choose RSA / .pem → save MyKeyPair.pem

AWS CLI

# create and save the key pair (example)
aws ec2 create-key-pair --key-name MyKeyPair --query 'KeyMaterial' --output text > MyKeyPair.pem
chmod 400 MyKeyPair.pem

2) Security Group — minimum rules

# Example (console): create a security group and add inbound rules
# Inbound:
#  - SSH (TCP 22)  — Source: your IP (recommended) or 0.0.0.0/0 (less secure)
#  - HTTP (TCP 80) — Source: 0.0.0.0/0
#  - HTTPS (TCP 443) — Source: 0.0.0.0/0

Tip: Restrict SSH to your IP address for better security. Use the console's "My IP" option.

3) Launch an Ubuntu instance

Console

EC2 → Launch instance → Choose Ubuntu Server (latest LTS) AMI → Instance type (t2.micro for free tier) → Key pair: MyKeyPair → Security group: choose the group you created → Launch

AWS CLI (example)

# Replace AMI-ID, subnet and security-group id with your values
aws ec2 run-instances \
  --image-id ami-0abcdef1234567890 \
  --count 1 \
  --instance-type t2.micro \
  --key-name MyKeyPair \
  --security-group-ids sg-0123456789abcdef0 \
  --tag-specifications 'ResourceType=instance,Tags=[{Key=Name,Value=Ubuntu-EC2}]' \
  --associate-public-ip-address

4) Connect via SSH

# make pem file private (if not already)
chmod 400 MyKeyPair.pem

# SSH into the instance (Ubuntu AMIs use user 'ubuntu')
ssh -i "MyKeyPair.pem" ubuntu@

If you receive a permission error, ensure the .pem file is chmod 400 and you are using the correct username (ubuntu, ec2-user, or ubuntu depending on AMI).

5) Basic instance interaction & commands

# update package list and upgrade
sudo apt update && sudo apt upgrade -y

# install nginx (example web server)
sudo apt install nginx -y
sudo systemctl enable --now nginx

# create a simple web page
sudo bash -c 'echo "<h1>Hello from EC2</h1>" > /var/www/html/index.html'

# check status and logs
sudo systemctl status nginx
sudo journalctl -u nginx --no-pager --since "1 hour ago"

# common useful commands
whoami
hostname -I
uptime
free -h
lsblk

To copy a local file to the instance (from your machine):

# from your local terminal (not the EC2 machine)
scp -i MyKeyPair.pem ./localfile.html ubuntu@:/home/ubuntu/

Troubleshooting tips

  • If SSH times out — check security group inbound rule for port 22 and ensure the instance has a public IP.
  • If connection refused — confirm the instance is running and the right username is used.
  • If HTTP not reachable — ensure port 80 open in SG and your web server (nginx) is running.

Deliverables Checklist

  • ✅ Launched EC2 Instance
  • ✅ Configured Security Group
  • ✅ Generated Key Pair
  • ✅ Successful SSH Connection
  • ✅ Basic instance interaction
